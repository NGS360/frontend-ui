# GitHub Actions Workflow for Frontend Tests

This file should be saved as `.github/workflows/frontend-tests.yml` in your project root.

```yaml
name: Frontend Tests

on:
  pull_request:
    paths:
      - 'frontend-ui/**'
      - '.github/workflows/frontend-tests.yml'
  push:
    branches:
      - main
      - develop
    paths:
      - 'frontend-ui/**'

jobs:
  test:
    name: Run Tests
    runs-on: ubuntu-latest
    
    defaults:
      run:
        working-directory: ./frontend-ui

    strategy:
      matrix:
        node-version: [20.x]

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js ${{ matrix.node-version }}
        uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node-version }}
          cache: 'npm'
          cache-dependency-path: frontend-ui/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Run linter
        run: npm run lint
        continue-on-error: false

      - name: Run format check
        run: npm run format -- --check
        continue-on-error: false

      - name: Run type check
        run: npx tsc --noEmit

      - name: Run tests
        run: npm test

      - name: Generate coverage report
        run: npm run test:coverage

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v4
        with:
          files: ./frontend-ui/coverage/lcov.info
          flags: frontend
          name: frontend-coverage
          fail_ci_if_error: false
        env:
          CODECOV_TOKEN: ${{ secrets.CODECOV_TOKEN }}

      - name: Comment coverage on PR
        if: github.event_name == 'pull_request'
        uses: davelosert/vitest-coverage-report-action@v2
        with:
          working-directory: ./frontend-ui

  build:
    name: Build Check
    runs-on: ubuntu-latest
    needs: test
    
    defaults:
      run:
        working-directory: ./frontend-ui

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20.x'
          cache: 'npm'
          cache-dependency-path: frontend-ui/package-lock.json

      - name: Install dependencies
        run: npm ci

      - name: Build application
        run: npm run build

      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          name: frontend-build
          path: frontend-ui/dist/
          retention-days: 7
```

## Environment Variables

If your tests need environment variables, add them to the workflow:

```yaml
- name: Run tests
  run: npm test
  env:
    VITE_API_BASE_URL: http://localhost:8000
    NODE_ENV: test
```

## Secrets Configuration

Add these secrets to your GitHub repository (Settings → Secrets and variables → Actions):

1. `CODECOV_TOKEN` - Token from codecov.io for coverage reporting (optional)

## Branch Protection Rules

Recommended branch protection rules for `main` branch:

1. Require status checks before merging:
   - Run Tests
   - Build Check
2. Require branches to be up to date before merging
3. Include administrators in these rules

## Caching Strategy

The workflow uses GitHub Actions cache for:
- Node modules (`npm ci` with cache)
- Build artifacts

This significantly speeds up subsequent runs.

## Notifications

To get notifications on test failures, configure GitHub notifications:
1. Go to repository Settings → Notifications
2. Enable "Actions" notifications
3. Configure your personal notification preferences

## Advanced Configuration

### Run on Schedule

Add scheduled test runs:

```yaml
on:
  schedule:
    # Run every day at 2 AM UTC
    - cron: '0 2 * * *'
```

### Matrix Testing

Test against multiple Node versions:

```yaml
strategy:
  matrix:
    node-version: [18.x, 20.x, 22.x]
```

### Parallel Jobs

Run linting and testing in parallel:

```yaml
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      # ... lint steps

  test:
    runs-on: ubuntu-latest
    steps:
      # ... test steps
```

## Troubleshooting

### Tests fail in CI but pass locally

- Check Node version matches
- Verify environment variables
- Check for timing issues (increase timeouts)

### Coverage upload fails

- Verify `CODECOV_TOKEN` is set
- Check file paths in coverage config
- Try running with `fail_ci_if_error: false`

### Slow CI runs

- Review which dependencies are cached
- Consider splitting jobs
- Use `npm ci` instead of `npm install`
